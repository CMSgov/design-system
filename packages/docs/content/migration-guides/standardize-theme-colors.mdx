With the expansion of the design system themes to include a fourth theme (CMS.gov), we noticed theme variables were inconsistent. In some cases colors and whole color palettes weren't being used and needed to be removed. We've cleaned up and reorganized these colors using design tokens and theming to provide a more intentional approach to color.

## Brand colors

We've updated the `primary` color to complete the spectrum from lightest to darkest.

### Alert 

The alert component has been updated to use `info` colors for the left border color and background color.

### Calendar picker button

Previously, `gray-cool-light` was mapped to the same system color token for all themes.

When removing `gray-cool-light`, we remapped the background color for the hover state of [calendar picker](/components/date-field/date-picker/) buttons to `primary-lightest`. On all but the core theme, this results in a slight color shift for the background color of those day-picker buttons when hovered.
