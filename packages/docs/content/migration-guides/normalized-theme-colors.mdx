intro: We've updated the theme color variables to exclude several families of non-semantic color variables and generally normalize the use of color variables across themes
---

import ColorMigrationTable from '../../src/components/content/ColorMigration/ColorMigrationTable.tsx';

Green was previously used in the `core` and `healthcare` themes for button variations, but it was unused in other
themes. The fact that green was being used for buttons really meant that green is a
secondary brand color for those themes. However, the blue info color for both those themes
was coming from their secondary color, which meant we needed to give that family of blues
a new home in the `info` status color.
